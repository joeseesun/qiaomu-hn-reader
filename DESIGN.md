# 乔木 HN 速读 Design System

## 1. 视觉主题与氛围

读取为：面向中文开发者和产品人的 Hacker News 每日阅读工具，用克制的暖白笔记阅读舱语言，倾向 Linear / Bear 的「柔和无衬线 + 笔记卡」家族，密度向阅读优先。

选定方向：C · 笔记应用。打开即刷——极简单行顶栏 + sticky 内容菜单 + 第一条新闻触手可及。每条新闻严格 1:1 中英文交替行：中文主行清晰、英文副行淡灰。不展示任何 API、数据源、翻译模型名等内部信息。

## 2. 色板与角色

- 页面底色：`#faf9f7`，暖白纸面，不用纯白也不用暖橙。
- 卡片面：`#ffffff`，纯白浮起，靠柔和阴影区分而非硬边框。
- 主文字：`#232323`，温暖近黑，不用纯黑。
- 次级文字：`#6e6a63`，用于英文副行、domain、时间。
- 三级文字：`#a39e95`，用于极弱信息。
- 主要重音：`#d4661a`，暖橙，只用于品牌点、活跃菜单、points 胶囊、链接 hover。
- 重音浅底：`rgba(212, 102, 26, 0.10)`，points 胶囊背景。
- 分隔/边框：`rgba(0, 0, 0, 0.05)`，极淡。
- 阴影：`0 1px 2px rgba(70, 55, 35, 0.04), 0 10px 24px rgba(70, 55, 35, 0.04)`，染暖色色调，极轻。

## 3. 排版规则

- 中文正文/UI 使用系统中文栈（PingFang SC / Microsoft YaHei 等），零下载字体。
- 标题/英文用 `-apple-system / SF Pro Text` 无衬线，零下载。
- 数字一律 `font-variant-numeric: tabular-nums`。
- 中文标题字重 600，正文 400/500；不用斜体，不用负字距。
- 中文行高 1.5-1.6；新闻标题行高 1.5。
- 中西文之间、中文与数字之间加空格（盘古之白）。

## 4. 组件样式

- 顶部品牌是阅读壳的一部分：品牌点 + 名称一行，右侧刷新；单行、低高度、融入背景，不跟随滚动。
- 内容菜单是顶栏下方的 sticky 横向文字菜单：首页精选、升温榜、产品雷达、收藏；滚动时只让这行保留。
- 新闻条目是笔记卡：纯白底、8px 圆角、柔和阴影；序号小字 + 中文主行 + 英文副行 + 摘要 + 单行元信息。
- 每条新闻严格两行标题：先中文（16px / 600），再英文（13px / 400 / 三级灰）。
- 不使用 API 入口卡、signal-board、endpoint grid 等内部信息组件。
- 筛选区只保留热度排序，放在搜索栏旁，不常驻大版面。
- 已读只由打开标题、来源、评论或速读等明确行为触发，并按帖子 ID 保存在本地；滚动不算已读。
- 已读只降低序号、标题和摘要的层级，不降低卡片、操作按钮或评论内容的透明度。
- 元信息按热度、评论数、域名、讨论速读、最佳评论、时间排列；讨论入口紧跟域名，窄屏自然换行。
- 卡片非功能控件区域点击后在新窗口打开原文；链接、收藏、讨论入口、评论内容和文本选择不触发整卡跳转。

## 5. 布局原则

- 首屏优先级：顶栏 / 内容菜单 / 搜索筛选 / 第一条新闻。禁止 hero 把 feed 推到首屏之外。
- 桌面与移动端都使用单列阅读流，居中容器 `max-width: 720px`。
- 第一条新闻必须在首屏可见区域。
- 移动端顺序与桌面一致，仅间距和字号收敛。

## 6. 深度层级

- 背景纯净暖白，不加纹理网格、不加渐变光晕。
- 笔记卡靠柔和双层阴影（近 + 远）浮起，不用重阴影。
- hover 时卡片边框轻微出现，强化可点击感。
- 无 backdrop-filter 玻璃拟态滥用。

## 7. Do's / Don'ts

- Do：严格一行中文一行英文，降低翻译不信任。
- Do：让 points、comments、domain 一眼可扫，但不喧宾夺主。
- Do：保留暖橙重音，只用于活跃态和热度。
- Don't：在页面展示 API endpoint、数据源、翻译模型名等内部信息。
- Don't：大标题营销化、居中 hero、低对比透明卡片。
- Don't：常驻「已载入」「正在读取」等系统自言自语文案。
- Don't：默认添加打赏、公众号、个人 Profile 等推广区块。

## 8. 响应式

- 单列阅读流，桌面 `max-width: 720px` 居中，移动端 `width: 100% - 24px`。
- 内容菜单在窄屏横向滚动，不换行撑破。
- 筛选区在移动端折叠为紧凑单行。
- 所有 flex/grid 文本子项加 `min-width: 0`，防英文长标题撑破。
- 顶部栏目和讨论内容使用标准 tab 语义，支持方向键、Home、End，并保持清晰焦点。
- 加载状态通过 `aria-busy` 和隐藏状态区播报，不让整条新闻流反复进入 live region。

## 9. Motion 哲学

- 动效强度 1/10：阅读列表以稳定为先，只保留必要的状态反馈。
- 卡片入场用 stagger（40-60ms 交错），从 `translateY(8px) + opacity:0` 入场，200ms ease-out。
- 卡片 hover 只轻微加深阴影，不发生位移或缩放。
- 按钮按压 `scale(0.97)`，120ms。
- 内容菜单切换、按钮状态有 150ms 颜色过渡。
- 骨架屏 shimmer 轻微。
- 支持 `prefers-reduced-motion`，降级时所有动效缩到 1ms。
